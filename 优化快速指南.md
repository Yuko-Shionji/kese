### 二、JdbcHelper 使用

#### 简化查询示例

**优化前（代码冗长）：**
```java
public User findById(Integer id) throws Exception {
    String sql = "SELECT * FROM users WHERE user_id = ?";
    Connection conn = null;
    PreparedStatement pstmt = null;
    ResultSet rs = null;
    
    try {
        conn = DBUtil.getConnection();
        pstmt = conn.prepareStatement(sql);
        pstmt.setInt(1, id);
        rs = pstmt.executeQuery();
        
        if (rs.next()) {
            User user = new User();
            user.setUserId(rs.getInt("user_id"));
            user.setUsername(rs.getString("username"));
            // ... 更多字段
            return user;
        }
        return null;
    } finally {
        // 关闭资源
        if (rs != null) rs.close();
        if (pstmt != null) pstmt.close();
        if (conn != null) conn.close();
    }
}
```

**优化后（简洁优雅）：**
```java
public User findById(Integer id) throws Exception {
    String sql = "SELECT * FROM users WHERE user_id = ?";
    
    return JdbcHelper.executeQuery(sql, rs -> {
        if (rs.next()) {
            User user = new User();
            user.setUserId(rs.getInt("user_id"));
            user.setUsername(rs.getString("username"));
            // ... 更多字段
            return user;
        }
        return null;
    }, id);
}
```

**效果：代码减少 60%！**

#### 简化更新示例

**优化前：**
```java
public int update(User user) throws Exception {
    Connection conn = null;
    PreparedStatement pstmt = null;
    
    try {
        conn = DBUtil.getConnection();
        String sql = "UPDATE users SET ... WHERE user_id = ?";
        pstmt = conn.prepareStatement(sql);
        pstmt.setString(1, user.getUsername());
        // ... 更多参数
        pstmt.setInt(10, user.getUserId());
        return pstmt.executeUpdate();
    } finally {
        if (pstmt != null) pstmt.close();
        if (conn != null) conn.close();
    }
}
```

**优化后：**
```java
public int update(User user) throws Exception {
    String sql = "UPDATE users SET ... WHERE user_id = ?";
    Object[] params = {
        user.getUsername(),
        // ... 更多参数
        user.getUserId()
    };
    
    return JdbcHelper.executeUpdate(sql, params);
}
```

**效果：代码减少 70%！**

### 三、BaseDAOImpl 使用

#### 快速实现 DAO

**步骤 1：继承 BaseDAOImpl**

```java
public class UserDAO extends BaseDAOImpl<User> {
    
    @Override
    protected String getTableName() {
        return "users";
    }
    
    @Override
    protected User mapResultSetToEntity(ResultSet rs) throws SQLException {
        User user = new User();
        user.setUserId(rs.getInt("user_id"));
        user.setUsername(rs.getString("username"));
        user.setRealName(rs.getString("real_name"));
        // ... 映射所有字段
        return user;
    }
    
    // 可选：重写钩子方法
    @Override
    protected String getPrimaryKeyColumn() {
        return "user_id";
    }
}
```

**步骤 2：使用继承的方法**

```java
UserDAO userDAO = new UserDAO();

// 直接使用继承的方法
User user = userDAO.findById(1);
List<User> all = userDAO.findAll();
List<User> page = userDAO.findByPage(1, 10);
int count = userDAO.getCount();

// 插入和删除
userDAO.insert(newUser);
userDAO.delete(userId);
```

**效果：无需重复编写 CRUD 代码！**

### 四、密码加密使用

#### 自动加密（推荐）

系统已自动处理密码加密，无需手动操作：

```java
UserService userService = new UserService();

// 添加用户时自动加密
User user = new User();
user.setUsername("newuser");
user.setPassword("123456"); // 明文输入
userService.addUser(user, "admin"); // 自动 MD5 加密存储

// 登录时自动验证加密
User loggedIn = userService.login("newuser", "123456"); // 自动加密后比对

// 修改密码时自动加密
userService.changePassword(userId, "oldpwd", "newpwd"); // 新旧密码都加密
```

#### 现有用户密码迁移

**方案 1：SQL 批量更新（推荐）**

```sql
-- 执行一次 SQL，批量加密现有用户密码
UPDATE users 
SET password = MD5(password) 
WHERE LENGTH(password) < 32;
```

**方案 2：首次登录时升级**

在登录逻辑中添加检测：

```java
public User login(String username, String password) throws Exception {
    User user = userDAO.login(username, encryptPassword(password));
    
    // 如果密码长度小于 32，说明是明文，升级为加密
    if (user != null && user.getPassword().length() < 32) {
        user.setPassword(encryptPassword(user.getPassword()));
        userDAO.update(user);
    }
    
    return user;
}
```

## 🔧 常见问题

### Q1: 日志文件在哪里？
**A:** 在项目根目录的 `logs/` 文件夹下，按日期分割：
```
kese1/
└── logs/
    ├── log_2026-03-19.txt
    ├── log_2026-03-20.txt
    └── ...
```

### Q2: 如何关闭日志输出？
**A:** 调用禁用方法：
```java
SimpleLogger.disable(); // 完全关闭日志
```

### Q3: 生产环境如何设置日志级别？
**A:** 在启动时设置：
```java
SimpleLogger.setLogLevel(SimpleLogger.Level.WARN); // 只记录警告和错误
```

### Q4: JdbcHelper 性能如何？
**A:** 与原生 JDBC 性能相同，只是封装了资源管理，无额外开销。

### Q5: BaseDAOImpl 能处理复杂查询吗？
**A:** BaseDAOImpl 提供基础 CRUD，复杂查询仍需自定义方法：
```java
public class UserDAO extends BaseDAOImpl<User> {
    // 基础 CRUD 已继承
    
    // 自定义复杂查询
    public List<User> findByCondition(String keyword, Integer status) {
        // 自定义实现
    }
}
```

### Q6: 密码加密后能解密吗？
**A:** 不能也不应该！MD5 是单向加密，只能验证无法解密。如需找回密码，应使用重置功能。

### Q7: 现有用户密码怎么办？
**A:** 执行 SQL 批量更新或首次登录时自动升级（见上方「现有用户密码迁移」）。

1. **日志记录**
   ```java
   // 记录关键业务操作

   // 不记录敏感信息
   ```

2. **数据库操作**
   ```java
   // 使用参数化查询
   JdbcHelper.executeQuery(sql, handler, params);
   
   // 避免字符串拼接
   // ❌ String sql = "SELECT * FROM t WHERE name='" + name + "'";
   ```

3. **密码处理**
   ```java
   // 让 Service 层自动加密
   userService.addUser(user, operator);

   ```

```java
// ❌ 禁止：字符串拼接 SQL
String sql = "SELECT * FROM users WHERE name='" + name + "'";

user.setPassword(password);
```
