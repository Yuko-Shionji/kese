# 快速配置与运行指南

## 🚀 快速开始（3 步完成）

### 第 1 步：配置数据库密码 ⚙️

打开文件 `config/database.properties`，修改密码：

```properties
# MySQL 密码（必须修改为实际密码）
db.password=你的数据库密码
```

**或者**如果你使用空密码，保持默认即可。

### 第 2 步：编译项目 🔨

在 PowerShell 中执行以下命令：

```powershell
cd E:\kese1

# 一键编译所有代码
javac -d bin -cp "lib/*" src/com/ruralwater/entity/*.java
javac -d bin -cp "lib/*" src/com/ruralwater/util/*.java  
javac -d bin -cp "lib/*;bin" src/com/ruralwater/dao/*.java
javac -d bin -cp "lib/*;bin" src/com/ruralwater/service/*.java
javac -d bin -cp "lib/*;bin" src/com/ruralwater/ui/*.java
javac -d bin -cp "lib/*;bin" src/com/ruralwater/*.java
```

### 第 3 步：测试数据库连接 ✅

```powershell
# 运行测试程序

```

## 🔧 常见问题解决

### Q1: 数据库连接失败
**错误信息：** `Access denied for user 'root'@'localhost'`

**解决方法：**
1. 检查 `config/database.properties` 中的密码配置
2. 确认 MySQL 服务已启动
3. 验证数据库 `rural_water_db` 是否存在

### Q2: 编译错误
**错误信息：** 找不到符号

**解决方法：**
```powershell
# 清除旧的 class 文件
Remove-Item bin\com\ruralwater\*.class -Recurse -ErrorAction SilentlyContinue

# 重新编译
cd E:\kese1
javac -d bin -cp "lib/*" src/com/ruralwater/entity/*.java
# ... 依次编译其他层
```

### Q3: 表不存在
**错误信息：** Table doesn't exist

**解决方法：**
```sql
-- 执行数据库初始化脚本
mysql -u root -p < database/schema.sql
```

## 💡 使用技巧

### 快捷操作
1. **快速刷新**：点击各面板的"刷新"按钮
2. **条件查询**：设置查询条件后点击"查询"
3. **查看详情**：双击表格行或选中后点击"查看详情"

1. **检测记录**：建议先选择水厂，再填写检测项
2. **审核操作**：只有待审核状态的记录才能审核
3. **设备管理**：及时更新设备状态（正常/维护/故障）
