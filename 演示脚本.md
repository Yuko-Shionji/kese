#### 5. 跨表关联查询
```sql
SELECT 
    wr.record_id,
    wp.plant_name,
    ws.source_name,
    u.real_name as tester_name,
    wr.sample_time,
    wr.conclusion
FROM water_quality_records wr
INNER JOIN water_plants wp ON wr.plant_id = wp.plant_id
INNER JOIN water_sources ws ON wp.source_id = ws.source_id
INNER JOIN users u ON wr.tester_id = u.user_id
WHERE wr.review_status = 'approved'
ORDER BY wr.sample_time DESC
LIMIT 0, 10;
```

## 常见问题准备

### Q1: 为什么用 C/S 架构而不是 B/S？
A: 课程设计要求使用 JavaSE，且 C/S 架构更适合桌面应用，响应速度快，交互性好。

### Q2: 事务是如何处理的？
A: 在 DAO 层通过 Connection 的 setAutoCommit(false) 开启事务，成功后 commit，异常时 rollback。

### Q3: 如何防止 SQL 注入？
A: 全部使用 PreparedStatement，通过参数绑定而非字符串拼接。

### Q4: 触发器的作用是什么？
A: 当插入不合格的检测数据时，自动生成预警记录，无需人工干预。

### Q5: 存储过程的优点？
A: 预编译、执行效率高、减少网络传输、便于维护复杂业务逻辑。
