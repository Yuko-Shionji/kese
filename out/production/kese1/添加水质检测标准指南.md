# 水质检测标准数据添加指南

## 问题说明
水质检测详情栏无法填写的原因是：**数据库中没有水质检测标准数据**。

在新增或编辑水质检测记录时，系统会从 `water_quality_standards` 表中加载检测标准（如 pH 值、浊度、余氯等），如果该表为空，则无法显示检测项目供用户填写。

## 解决方案

### 方法一：使用批处理文件（推荐）
1. 确保 MySQL 服务已启动
2. 双击运行 `init_standards.bat`
3. 等待执行完成，看到"✓ 数据初始化成功"提示

### 方法二：手动执行 SQL 脚本
打开命令行，执行以下命令：
```bash
mysql -hlocalhost -uroot -p1234 keshe < database\init_water_quality_standards.sql
```

### 方法三：使用数据库管理工具
1. 打开 Navicat、MySQL Workbench 等工具
2. 连接到 `keshe` 数据库
3. 打开并执行 `database/init_water_quality_standards.sql` 文件

## 验证数据是否添加成功

### 方式 1：运行测试程序
```bash
java -cp "out\production\kese1;lib\*" com.ruralwater.TestStandardsData
```

### 方式 2：直接查询数据库
```sql
SELECT category, COUNT(*) as count 
FROM water_quality_standards 
WHERE is_active = 1 
GROUP BY category;
```

预期结果：
- physical (物理指标): 6 项
- chemical (化学指标): 20 项  
- biological (生物指标): 5 项
- radiological (放射性指标): 2 项

### 方式 3：在系统中查看
1. 启动系统
2. 进入"水质检测管理"
3. 点击"新增记录"
4. 应该能看到检测详情表格中自动填充了所有检测标准

## 常见问题

### Q1: 执行 SQL 脚本报错
**解决：** 检查以下几点
- MySQL 服务是否启动
- 数据库 `keshe` 是否存在
- 用户名密码是否正确（配置文件：`config/database.properties`）

### Q2: 仍然无法填写检测详情
**解决：** 
1. 重启系统
2. 查看控制台是否有错误信息
3. 确认 `water_quality_standards` 表中有数据且 `is_active=1`

### Q3: 需要添加自定义检测项
**解决：** 
可以直接在数据库中插入：
```sql
INSERT INTO water_quality_standards 
(item_name, item_code, unit, max_value, category, description, is_active) 
VALUES 
('自定义检测项', 'CUSTOM001', 'mg/L', 10.00, 'chemical', '说明', 1);
```

## 技术说明

### 数据库表结构
```sql
CREATE TABLE water_quality_standards (
    standard_id INT PRIMARY KEY AUTO_INCREMENT,
    item_name VARCHAR(100) NOT NULL,        -- 检测项名称
    item_code VARCHAR(20) UNIQUE,           -- 检测项编码
    unit VARCHAR(20),                       -- 单位
    standard_value DECIMAL(10, 3),          -- 标准值
    min_value DECIMAL(10, 3),               -- 最小值
    max_value DECIMAL(10, 3),               -- 最大值
    category ENUM('physical', 'chemical', 'biological', 'radiological'),
    description TEXT,                       -- 描述
    is_active TINYINT DEFAULT 1             -- 是否启用
);
```

### 代码中的使用
在 `AddRecordDialog.loadQualityStandards()` 方法中：
- 查询所有 `is_active = 1` 的检测标准
- 填充到检测详情表格
- 用户只需填写实测值和选择是否合格
